# CuraLive AI Management Platform — Grok Brief

## Executive Summary

CuraLive is a simplified AI feature management platform designed to help organizations navigate 28 AI-powered features without overwhelming users. The platform uses a bundle-based approach, guided onboarding, and progressive feature unlocking to maximize adoption, reduce decision paralysis, and accelerate time-to-value.

**Core Philosophy:** Start simple, unlock gradually, discover naturally.

---

## Platform Overview

### What is CuraLive?

CuraLive is an intelligent event management platform that bundles 28 AI features into 5 role-based bundles, each designed to solve specific business problems. Instead of presenting all features at once, CuraLive guides users through a personalized discovery journey:

1. **Onboarding Quiz** (2-3 minutes) → Recommends the right bundle
2. **Progressive Unlock** (Day 1 → Week 1 → Month 1) → Introduces features gradually
3. **AI Shop** (Marketplace) → Helps users discover and activate features
4. **Tracking & Analytics** → Measures ROI and engagement

### Why This Approach?

**Problem:** Organizations with many AI features face "decision paralysis" — users don't know which features to use, when to use them, or how they benefit from them.

**Solution:** CuraLive reduces complexity through:
- **Bundling** - Group 28 features into 5 role-based bundles (6-7 features each)
- **Guidance** - Quiz recommends the right bundle in 2-3 minutes
- **Gradual Unlock** - Introduce 3 features on Day 1, 4-5 on Week 1, full bundle on Month 1
- **Discovery** - AI Shop marketplace helps users find features they didn't know existed
- **Measurement** - Track adoption, usage, and ROI for each feature

## How the Platform Works: User Journey

### Step 1: Onboarding Quiz (2-3 minutes)

**Purpose:** Recommend the right bundle based on user's role, challenges, and needs

**The 4 Questions**
1. **What's your primary role?** (IR, Compliance, Operations, Marketing, Executive, Other)
2. **What's your biggest challenge?** (Follow-up, Compliance, Operations, Content, ROI, Multiple)
3. **What types of events do you run?** (Earnings, Board, Webinar, Conference, Sales, Multiple)
4. **What's your budget?** (Starter $299, Pro $899, Enterprise custom, Unsure)

**Quiz Logic**
- Answers map to bundle recommendations
- Example: IR Manager + Follow-up challenge + Earnings calls + Pro budget → Bundle A
- Example: Compliance Officer + Compliance challenge + Board meetings + Enterprise budget → Bundle E

**Results Page**
- Recommended bundle with description
- 3 "quick win" features to activate immediately
- Secondary bundle suggestions
- Pricing tier and next steps

### Step 2: Progressive Feature Unlock (Day 1 → Month 1)

**Purpose:** Introduce features gradually to maximize adoption and reduce overwhelm

**Day 1: Quick Start (3 Features)**
- Activate 3 core features immediately
- Show instant value (run a test event)
- Send welcome email with quick start guide
- Example for Bundle A: Sentiment Dashboard, Event Summaries, Auto Follow-Up

**Week 1: Expansion (4-5 More Features)**
- Unlock triggered by: 7 days passed OR first event completed (whichever comes first)
- Recommend features based on usage patterns
- Send email: "You're ready for more! Here are 4 new features"
- Example for Bundle A: Q&A Deep-Dive, Investor Mood Scoring, Pre/Post-Event Packs

**Month 1: Full Bundle Access (Remaining Features)**
- Unlock triggered by: 30 days passed OR 2+ events completed (whichever comes first)
- Complete bundle access with dedicated training
- Offer 15-minute onboarding call
- Example for Bundle A: Advanced Analytics, Custom Workflows, API Access

**Ongoing: Add-On Opportunities**
- Suggest complementary bundles ("Try Compliance free for 30 days")
- Offer advanced features from other bundles
- Provide case studies of similar users

### Step 3: AI Shop (Feature Discovery Marketplace)

**Purpose:** Help users discover features they didn't know existed

**Shop Pages**

1. **Shop Home (Discovery Hub)**
   - Search bar with auto-suggest
   - Featured features (3-4 rotating weekly)
   - Recommended for you (4-6 personalized features)
   - Browse by category (6 categories)
   - Browse by problem ("How do I...")

2. **Category Browse**
   - All features in category (e.g., Analytics, Engagement, Compliance)
   - Sort by: Relevance, Most Popular, Newest, A-Z
   - Filter by: My Bundle, All Features, Activated, Locked
   - Feature grid with cards

3. **Feature Detail Page**
   - Feature name, description, icon
   - Key benefits with metrics
   - Common use cases
   - Pricing & availability
   - Complementary features
   - User reviews (4.8/5 stars, 247 reviews)
   - Call-to-action buttons

**Feature Cards**
- Status badge (Available, Locked, Activated)
- Time saved metric (e.g., "2 hours/event")
- Star rating and review count
- Key benefit highlight
- One-click activation button
- "Learn More" link

**Recommendation Algorithm**
- Input signals: Bundle, role, activated features, usage patterns, industry, event types
- Ranking: Complementary (40%) + Role popularity (30%) + Satisfaction (20%) + Trending (10%)
- Returns top 6 recommendations

### Step 4: Tracking & Analytics

**Purpose:** Measure adoption, usage, and ROI

**Key Metrics Tracked**
- Quiz completion rate (% of users who complete)
- Bundle adoption rate (% of users per bundle)
- Feature activation rate (% of features activated per bundle)
- Time-to-value (days until user sees ROI)
- Feature usage (frequency and duration)
- Feature satisfaction (1-5 rating)
- Churn rate (% of users who cancel)
- Upgrade rate (% moving from Starter to Pro)
- Revenue per user (ARPU)

**Analytics Dashboard**
- Unlock funnel (% reaching each phase)
- Feature adoption curve
- Time-to-value metrics
- Engagement by feature
- Churn by unlock phase
- Revenue impact

## Technical Architecture

### Frontend Components

**1. Onboarding Quiz** (React)
- 4-question interactive form
- Progress bar and question counter
- Conditional logic for recommendations
- Results page with quick wins
- Feature activation buttons

**2. Progressive Unlock System** (TypeScript Backend)
- Feature unlock engine
- Time-based triggers (Day 1, Week 1, Month 1)
- Usage-based triggers (events, activations)
- Hybrid triggers (whichever comes first)
- Feature state management
- Usage tracking
- Satisfaction rating system

**3. AI Shop** (React)
- Feature marketplace UI
- Search and filtering
- Category browsing
- Problem-solving discovery
- Feature cards with details
- Recommendation engine
- One-click activation

### Database Schema

```sql
-- User feature state tracking
CREATE TABLE user_feature_state (
  user_id VARCHAR(255),
  feature_id VARCHAR(255),
  bundle_id VARCHAR(255),
  status ENUM('locked', 'available', 'activated', 'disabled'),
  unlocked_at TIMESTAMP,
  activated_at TIMESTAMP,
  first_used_at TIMESTAMP,
  usage_count INT DEFAULT 0,
  satisfaction_rating INT,
  PRIMARY KEY (user_id, feature_id)
);

-- User progress tracking
CREATE TABLE user_progress_state (
  user_id VARCHAR(255),
  bundle_id VARCHAR(255),
  signup_date TIMESTAMP,
  first_event_date TIMESTAMP,
  events_completed INT DEFAULT 0,
  features_activated INT DEFAULT 0,
  current_phase ENUM('day1', 'week1', 'month1', 'ongoing'),
  last_unlock_date TIMESTAMP,
  PRIMARY KEY (user_id, bundle_id)
);

-- Feature catalog
CREATE TABLE features (
  feature_id VARCHAR(255) PRIMARY KEY,
  name VARCHAR(255),
  description TEXT,
  category VARCHAR(50),
  bundle_id VARCHAR(255),
  time_saved VARCHAR(50),
  rating DECIMAL(2,1),
  reviews INT
);
```

### Integration Points

**Quiz → Progressive Unlock**
- Quiz recommends bundle → Initialize features for that bundle
- Day 1 features auto-activate → Show in shop as "activated"

**Progressive Unlock → Shop**
- Week 1 features unlock → Show in shop as "available"
- Month 1 features unlock → Show in shop as "available"
- Locked features → Show unlock date

**Shop → Backend**
- Click "Activate" → Calls `activateFeature()`
- Feature usage → Calls `recordFeatureUsage()`
- Event completion → Calls `recordEventCompletion()`
