SECTION 2 — RUNTIME ARCHITECTURE
───────────────────────────────────

Backend framework: Express.js + tRPC (TypeScript)
Frontend framework: React 19 + Vite

API layer:
  tRPC handles all typed API calls at /api/trpc
  REST endpoints exist for webhooks and public-facing token routes
  WebSocket support via ws (used for real-time transcription and metrics)

Frontend serving (production):
  Node server uses express.static to serve dist/public/
  All unmatched routes fall through to the SPA's index.html
  SPA routing handled client-side by wouter (not React Router)

Frontend serving (development):
  Vite dev server middleware is attached directly to Express
  (server/_core/vite.ts registers Vite as middleware)

Routing model: Single Page Application (SPA)
  All navigation is client-side via wouter <Route> / <Switch>
  Server only intercepts /api/*, /health, and webhook routes

Real-time: Ably (pub/sub for live events), WebSocket for metrics

SECTION 4 — AI SERVICE INTEGRATION
─────────────────────────────────────

Python AI service location: curalive_ai_core/
  Structure: FastAPI app with api/, core/, db/, models/,
             schemas/, services/, utils/, workers/
  Database: Shares same PostgreSQL via DATABASE_URL
  Driver: psycopg v3 (psycopg[binary]==3.3.3)
  Driver string: postgresql+psycopg:// (auto-converted from postgresql://)
  LLM key: Reads as LLM_API_KEY (NOT OPENAI_API_KEY)

Connection from Node to Python:
  File: server/services/AICoreClient.ts
  Base URL: process.env.AI_CORE_URL ?? "http://localhost:5000"
  Timeout: 30s default, 120s for long-running operations
  Retries: 2 attempts with 1s delay
  Error types: timeout, network, upstream, parse

What Node sends to Python:
  Canonical event objects with segments, speakers, transcript
  Module list to execute (e.g. sentiment, compliance, summary)

Also referenced in:
  server/routers/socialMedia.ts (AI_CORE_URL for one endpoint)

SECTION 6 — BUILD & DEPLOYMENT LOGIC
───────────────────────────────────────

Dev command:
  NODE_ENV=development tsx watch server/_core/index.ts

Build command:
  NODE_OPTIONS='--max-old-space-size=4096' \
  vite build --logLevel warn && \
  esbuild server/_core/index.ts \
    --platform=node --packages=external --bundle --format=esm \
    --outdir=_build && \
  rm -rf dist && cp -r _build dist

What the build produces:
  _build/public/   -> React SPA (HTML, JS, CSS)
  _build/index.js  -> Node server bundle
  Then entire _build/ is copied to dist/

Production start:
  NODE_ENV=production node dist/index.js

Port binding: Reads $PORT env var, binds 0.0.0.0
  (defaults to 3000 if $PORT not set)

Static file serving (production):
  Express serves dist/public/ via express.static
  SPA fallback: all unmatched routes -> dist/public/index.html

Both dist/ and _build/ currently exist on disk (build has been run).

Python production start:
  uvicorn app.main:app --host 0.0.0.0 --port $PORT
  Working directory: curalive_ai_core/

SECTION 8 — CONSISTENCY ISSUES
─────────────────────────────────

ISSUE 1 — Duplicate App component
  App.tsx (active) and AppEnhanced.tsx (orphaned) both exist.
  AppEnhanced.tsx is not imported anywhere in the live app.
  Risk: Confusion about which is canonical. Low runtime risk, high
  maintenance risk.

ISSUE 2 — Migration numbering collision
  0030 and 0031 each have two SQL files with different names.
  Drizzle migration order depends on filename sort order.
  Risk: Ambiguous migration history; future migrations may misfire.

ISSUE 3 — REPLIT_DEV_DOMAIN still used in some routers
  Files still directly reading REPLIT_DEV_DOMAIN:
    server/routers/mailingListRouter.ts
    server/routers/socialMedia.ts
    server/routers/bridgeConsoleRouter.ts
    server/routers/shadowModeRouter.ts
    server/routers/operatorDashboardRouter.ts
  These will return empty strings or fallback values on Render where
  REPLIT_DEV_DOMAIN is not set. Mailing list unsubscribe links,
  social OAuth callbacks, shadow mode webhooks, and operator report
  URLs will be broken or malformed on Render.

ISSUE 4 — LLM key name mismatch between services
  Node reads: OPENAI_API_KEY
  Python reads: LLM_API_KEY
  Same key value, different variable name. Must set both on Render.
  Not a code bug but an operational risk — easy to misconfigure.

ISSUE 5 — AI_CORE_URL default points to localhost:5000
  In server/services/AICoreClient.ts:
    const AI_CORE_BASE_URL = process.env.AI_CORE_URL ?? "http://localhost:5000"
  On Render, services cannot reach each other via localhost.
  AI_CORE_URL must be set to the internal Render URL.
  If not set, all AI analysis will silently fail.

ISSUE 6 — 212 page files, many potentially unused
  The pages/ directory has grown to 212 files. Some are clearly
  legacy or experimental. No tree-shaking at the page level — all
  are imported in App.tsx, increasing initial bundle analysis cost.


SECTION 9 — RISK INDICATORS

RISK: DEPLOYMENT FAILURE
  [HIGH] AI_CORE_URL not set on Render -> Python service unreachable,
    all AI analysis calls fail silently
  [HIGH] LLM_API_KEY not set on Python service -> all LLM processing fails
  [MEDIUM] REPLIT_DEV_DOMAIN used in 5 routers -> malformed URLs on Render

RISK: RUNTIME ERRORS
  [MEDIUM] operatorDashboardRouter.ts ~line 363: report URLs built with
    REPLIT_DEV_DOMAIN — will produce empty/broken links on Render
  [MEDIUM] socialMedia.ts: OAuth redirect URI hardcoded to
    REPLIT_DEV_DOMAIN or localhost:5000 — social OAuth will fail on Render
  [MEDIUM] shadowModeRouter.ts: webhook URL construction relies on
    REPLIT_DEV_DOMAIN — Recall.ai webhooks will not register correctly

RISK: ENVIRONMENT MISMATCH
  [HIGH] Two env var names for the same OpenAI key (OPENAI_API_KEY vs
    LLM_API_KEY) — one will almost certainly be missed at deploy time
  [MEDIUM] APP_ORIGIN defaults to curalive-platform.replit.app —
    if not set explicitly on Render, CORS and webhook return URLs
    will point to wrong domain

RISK: BROKEN ROUTING
  [LOW] AppEnhanced.tsx exists but is unused — not a routing risk
    today but could cause confusion if accidentally wired in
  [LOW] Migration numbering collision — does not affect routing but
