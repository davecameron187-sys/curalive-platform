CuraLive — Replit Validation Brief
No GitHub Sync. Validation and Proof Only.
Purpose

Replit’s role is not to rebuild CuraLive, re-implement features, or do broad development work.

Replit’s role is to:

validate the current Shadow Mode + embedded live console experience,
execute the required validation tracks,
report real bugs,
make only small targeted fixes if a runtime issue blocks testing.

This keeps costs down and avoids duplicate work.

The current Manus handoff says the embedded live console, WebPhone integration, and staging fixes are complete and ready for validation.

Critical instruction
Replit must not:
redo Shadow Mode
rebuild the live console
recreate WebPhone integration
rewrite archive/export/handoff workflows
invent new architecture
do broad refactors
manually rebuild everything from briefs
Replit should only:
use the code/environment already available in Replit
validate the implemented experience
report real bugs
apply minimal fixes only when needed to complete validation

If a missing feature requires real build work, send it back to Manus.

No-GitHub rule

If GitHub is not accessible from Replit:

do not block on repo sync
do not attempt to recreate the whole branch manually
do not treat the brief as a rebuild spec

Instead:

validate what is already present in the Replit environment
compare behavior against the expected live console brief
report gaps as either:
runtime bug
missing code in environment
needs Manus sync/build

When Replit should fix something
Replit may fix directly only if:
it is a small runtime/UI/config issue
it blocks a validation track
the fix is narrow and low-risk
it does not require architectural rebuild work
Replit should send it back to Manus if:
it is missing implementation, not just a bug
it requires major code changes
it affects architecture or workflow design
it means rebuilding a feature already claimed complete
