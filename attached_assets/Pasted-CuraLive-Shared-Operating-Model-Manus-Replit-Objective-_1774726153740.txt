CuraLive Shared Operating Model — Manus + Replit
Objective

Keep costs down, move fast, and avoid duplicated work.

Core rule
Manus builds
Replit proves
GitHub is the source of truth
What Manus should do

Manus should handle the heavy lifting first:

architecture changes
backend logic
frontend feature implementation
routing
schema/database updates
fallback/provider logic
docs
test scaffolding
workflow completion before staging
Simple rule

If it is building or changing the product, do it in Manus first.

What Replit should do

Replit should be used only when the app needs to be run and verified in a live-like environment.

Replit should handle:

staging validation
integration testing
UI behavior checks
real environment debugging
operator acceptance testing
bug reproduction
final refinement where needed
Simple rule

If it is running, testing, or proving the product works, do it in Replit.

What stays in GitHub

Everything important must be reflected in GitHub:

implementation status
briefs
issues
PRs
bug reports
what is complete
what is still pending

If it is not in GitHub, it is not agreed.

Cost-saving rule

Do not use Replit for early feature construction if Manus can do it first.

Use Replit only for:

validation
environment-specific fixes
final polish

That keeps Replit focused on the highest-value work.

Current application of this rule
Manus should continue with:
remaining WebPhone-first work
webcast/audio integration work
fallback integration logic
any remaining feature completion
code-level fixes from staging feedback
Replit should focus on:
Track A–E staging tests
operator acceptance testing
webcast/audio validation
fallback behavior validation
real runtime bug discovery
final UI/workflow refinement only if needed
Delivery standard
Manus must return:
commit hash or PR
files changed
what was implemented
what is still incomplete
what needs Replit validation
Replit must return:
pass/fail
screenshots/logs/evidence
bugs with reproduction steps
severity
whether issue is code, config, or environment
Decision rule
If the problem is feature missing or logic incomplete → send it to Manus
If the problem is runtime behavior, staging failure, or environment issue → send it to Replit
