Phase 2: Identify Node → Python Service Communication

The Problem

On Replit, Node and Python run in the same process/environment. On Render, they are separate services with different URLs. The Node app must know how to reach the Python service.

What You Must Do

Task 2.1: Find Where Node Calls Python

Search the Node codebase for any HTTP calls to the Python service:

Bash


# Search for Python service calls
grep -r "localhost:8000\|localhost:5000\|python\|ai_core\|AI_CORE" server/ --include="*.ts" | head -50



Also search for environment variables that might configure this:

Bash


grep -r "PYTHON\|AI_CORE\|AI_SERVICE" server/ --include="*.ts" | head -50



Task 2.2: Document the Configuration

Find the exact code that calls the Python service and answer these questions:

1.
What is the current Python service URL? (e.g., http://localhost:8000 )

2.
Is it hardcoded or configurable via environment variable?

3.
What environment variable name is used? (e.g., PYTHON_AI_CORE_URL, AI_SERVICE_URL, etc.)

4.
What endpoints does Node call on the Python service? (e.g., /analyze, /report, etc.)

Example output format:

Plain Text


File: server/services/AIReportPipeline.ts
Line 45: const pythonUrl = process.env.PYTHON_AI_CORE_URL || "http://localhost:8000"
Endpoint called: POST /api/generate-report



Task 2.3: Confirm or Create Environment Variable

If Node calls the Python service, ensure there is an environment variable to configure it:

TypeScript


// Example pattern (find the actual code )
const PYTHON_AI_CORE_URL = process.env.PYTHON_AI_CORE_URL || "http://localhost:8000";



For Render, this will be set to the internal Python service URL (e.g., http://curalive-ai-core:10000 ).
