SERVER SIDE — Two REST endpoints in server/_core/index.ts:

Transcript Download (around line 628):

GET /api/archives/:id/transcript
Queries SELECT event_name, client_name, event_date, transcript_text FROM archive_events WHERE id = ?
Returns a .txt file with a header line (event name, client, date) followed by the full transcript
Content-Type: text/plain
Filename format: {EventName}_Transcript.txt
Recording Download (around line 651):

GET /api/archives/:id/recording
Queries SELECT event_name, recording_path FROM archive_events WHERE id = ?
Serves the MP3 file from uploads/recordings/ directory using res.sendFile()
Content-Type: audio/mpeg
Filename format: {EventName}_Recording.mp3
SERVER SIDE — Archive detail tRPC endpoint in server/routers/archiveUploadRouter.ts:

The getArchiveDetail procedure (around line 1175) returns two boolean flags:

has_transcript: true/false — based on whether transcript_text exists for that archive
has_recording: true/false — based on whether recording_path exists for that archive
The raw text and file path are NOT exposed to the client — only the boolean flags.
