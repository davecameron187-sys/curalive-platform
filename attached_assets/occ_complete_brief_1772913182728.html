📌 Executive Summary: The Operator Console (OCC) is the command centre for managing live events on the CuraLive platform. This comprehensive brief defines every aspect of the frontend interface, user interactions, data flow, and operational requirements to ensure a production-ready, enterprise-grade control centre.

🏗️ System Architecture

The Operator Console is built on a modern React frontend with real-time updates via Ably WebSocket and backend API calls via tRPC. The architecture supports 1000+ concurrent participants, 100+ active conferences, and sub-100ms real-time updates.

🎨 Frontend (React)

React 19 with TypeScript, Tailwind CSS, real-time state management via Context + useReducer

⚡ Real-Time (Ably)

WebSocket connections for sub-100ms participant status, sentiment, Q&A, transcription updates

🔌 Backend API (tRPC)

Type-safe RPC for initial data load, search, filters, actions, audit logging

💾 Database

MySQL/TiDB for conference data, participant state, audit trails, operator preferences

3. Live Call Counter Dashboard (8 Metric Cards)

Provides at-a-glance status of current event:

  • Active Calls: Number of running conferences (Blue)
  • Total Participants: Sum of all participants (Green)
  • Speaking Now: Participants currently speaking (Amber)
  • Muted Participants: Count of muted participants (Red)
  • Sentiment Score: Average sentiment gauge 0-100 (Dynamic)
  • Q&A Submitted: Pending questions count (Purple)
  • Transcription Status: Progress % and status (Cyan)
  • Recording Status: Active recording with timer (Red)

📊 Core Sections

Section 1: Conference Overview

Displays all running, pending, and completed conferences with quick access to controls.

Tabs:

  • Running: Active conferences (default view)
  • Pending: Upcoming conferences not yet started
  • Completed: Finished conferences (last 24 hours)
  • Alarms: Conferences with alerts (high sentiment drop, etc.)

Key Columns:

  • Call-ID, Subject, Reseller, Start Time, Duration, Participants
  • Mod Code, Part Code, Dial-In, Status, Actions
  • Actions: Open CCP, Split View, Details, Export

Section 2: Conference Control Panel (CCP)

Detailed controls and management for the selected conference.

Subsections:

  • Conference Bar: Record, Lock, Mute, Disconnect, Dial Out buttons
  • Search Bar: Real-time search by name, company, phone, location
  • Filter Bar: All, Mod, Part, Unmuted, Muted, Parked, Connected, Waiting, Web, Speak Req
  • Participant Table: Real-time participant list with multi-select
  • Multi-Select Actions: Bulk operations on selected participants

Section 3: Feature Tabs

Advanced features and detailed information:

  • Monitoring: Call quality metrics (bandwidth, latency, jitter, packet loss)
  • Connection: IP addresses, codec, encryption, NAT info
  • History: Participant previous calls and notes
  • Audio Files: Recording download, playback, transcription
  • Chat: Real-time chat transcript
  • Notes: Operator notes about participant or call
  • Q&A Queue: Submitted questions, voting, moderation
  • CuraLive Direct: Direct messaging to participant

✨ Feature Specifications

Conference Management

  • Create new conferences with event type, title, start time, duration
  • Edit conference details (title, description, settings)
  • Delete pending/completed conferences
  • Lock/unlock conferences (prevent new joins)
  • Start/stop recording with confirmation
  • Export conference data as CSV

Participant Management

  • Add participants via phone or email
  • Mute/unmute individual or bulk participants
  • Park/unpark participants (hold with music)
  • Disconnect participants with confirmation
  • Transfer participants to another operator
  • View participant history and notes

Search & Filtering

  • Real-time search across name, company, phone, location
  • Multiple active filters with AND logic
  • Save/load filter combinations
  • Filter counts for each category
  • Case-insensitive partial matching

Real-Time Monitoring

  • Call quality metrics (bandwidth, latency, jitter, packet loss)
  • Participant sentiment scores (0-100, color-coded)
  • Transcription status and progress
  • MOS (Mean Opinion Score) for call quality
  • Alerts for sentiment drops or quality issues

Q&A Management

  • Display Q&A queue with submitted questions
  • Approve/reject questions
  • Edit question text before approval
  • Pin important questions
  • View vote counts
  • Mark questions as answered
  • Archive answered questions

🔄 State Management

Use React Context + useReducer for global state with Ably for real-time updates. Memoize expensive components with React.memo, useMemo, useCallback. Use react-window for virtual scrolling of 1000+ participant rows.

🔒 Security & Compliance

  • OAuth 2.0 authentication via Manus
  • Role-based access control (RBAC)
  • Audit trail for all operator actions
  • HTTPS/TLS encryption for all API calls
  • Mask sensitive data (phone, email)
  • GDPR, CCPA, SOC 2, HIPAA compliance
  • Secure WebSocket (WSS) for Ably
