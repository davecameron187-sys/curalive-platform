What still needs to be completed

Even though the core console is in place, the remaining workflows need to be finished and hardened.

1. Real live transcript pipeline

The roadmap requires near-real-time transcript in the console. The current console fetches transcript segments from backend queries and marks transcript unavailable if the session is running but no segments exist. That is good UI behavior, but the live pipeline still needs to be fully operational end-to-end in production.

Required work
Verify Recall.ai bot join flow
Verify Recall webhook receipt
Verify transcript segment persistence
Verify Ably/live propagation into the console
Verify “live”, “stale”, and “unavailable” states behave correctly
Remove any remaining hidden mock/demo assumptions anywhere in the transcript path
Done means
real transcript appears in the console from live event input
stale/unavailable states are honest and recover correctly
no manual refresh required
2. Archive upload + transcription resilience

Archive uploads are currently failing when Whisper/OpenAI transcription hits 429 insufficient_quota. This must not break archive ingestion.

Required work
Decouple archive upload from transcription
Store archive first
Run transcription as a separate status-driven job
Handle insufficient_quota explicitly
Show clean transcription status in UI
Allow retry later without re-uploading archive
Required statuses
uploaded
transcription_pending
transcription_processing
transcription_complete
transcription_failed_quota
transcription_failed_provider
Done means
archive upload succeeds even if transcription fails
raw provider JSON is never shown to users
retry path exists
