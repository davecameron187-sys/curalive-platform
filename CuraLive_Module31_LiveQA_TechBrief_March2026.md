## 1. Executive Summary

Module 31 — the **Autonomous Live Q&A Intelligence Engine** — is fully built and integrated into Shadow Mode. It enables real-time attendee question submission during investor events, with AI-powered triage, multi-jurisdictional compliance screening, and operator management through a purpose-built console.

The operator never leaves the existing Shadow Mode console. A "Live Q&A" tab gives full control over every question with one-click actions, AGI-powered triage, compliance risk indicators, and auto-draft responses.

**Core stack**: React 19 + tRPC + Ably real-time + GPT-4o-mini + MySQL/Drizzle

## 4. Backend API — tRPC Router (`liveQa.*`)

**File**: `server/routers/liveQaRouter.ts` (388 lines)
**Auth**: All operator endpoints use `operatorProcedure` (requires `operator` or `admin` role). Public endpoints use `publicProcedure`.

### Operator Endpoints (require login + operator role)

| Endpoint | Type | Description |
|----------|------|-------------|
| `liveQa.createSession` | Mutation | Create Q&A session linked to Shadow Mode. Returns session with 8-char access code |
| `liveQa.getSession` | Query | Get session by ID |
| `liveQa.getSessionByShadow` | Query | Get Q&A session linked to a Shadow Mode session |
| `liveQa.updateSessionStatus` | Mutation | Set session to `active` / `paused` / `closed` |
| `liveQa.listQuestions` | Query | List questions with answer count + unresolved compliance flags. Filterable by status |
| `liveQa.updateQuestionStatus` | Mutation | Set question status + optional operator notes. Publishes Ably event |
| `liveQa.generateDraft` | Mutation | AI-generate a compliance-safe draft response (GPT-4o-mini) |
| `liveQa.submitAnswer` | Mutation | Submit operator response, mark as answered. Publishes Ably event |
| `liveQa.getAnswers` | Query | List all answers for a question |
| `liveQa.getComplianceFlags` | Query | List compliance flags for a question |
| `liveQa.resolveComplianceFlag` | Mutation | Mark a compliance flag as resolved |
| `liveQa.listSessions` | Query | List all Q&A sessions |

### Public Endpoints (no auth — attendee-facing)

| Endpoint | Type | Description |
|----------|------|-------------|
| `liveQa.getSessionByCode` | Query | Get session info by access code (limited fields) |
| `liveQa.submitQuestion` | Mutation | Submit question with AI triage. Auto-flags if compliance risk > 70 |
| `liveQa.listQuestionsPublic` | Query | List approved/triaged/answered questions (hides pending/rejected) |
| `liveQa.upvoteQuestion` | Mutation | Upvote with rate limiting (10s cooldown per fingerprint) |

### Real-Time (Ably)

**Channel**: `curalive-qa-{sessionId}`

| Event | Trigger | Payload |
|-------|---------|---------|
| `qa.submitted` | New question submitted | questionId, text, category, triageScore, complianceRiskScore, status |
| `qa.statusChanged` | Status updated (approve/reject/flag/answer) | questionId, newStatus, operatorNotes, timestamp |

## 5. AI Triage Service

**File**: `server/services/LiveQaTriageService.ts` (142 lines)
**Model**: GPT-4o-mini via `invokeLLM`

### `triageQuestion(questionText, eventName, clientName, existingQuestions)`

Analyses each submitted question and returns:

- **category**: financial / operational / esg / governance / strategy / general
- **triageScore**: 0–100 (urgency/relevance)
- **triageClassification**: high_priority / standard / low_priority / duplicate / hostile
- **triageReason**: Natural language explanation
- **complianceRiskScore**: 0–100 (regulatory risk)
- **priorityScore**: 0–100 (combined queue ordering)
- **complianceFlags[]**: Per-jurisdiction flags with risk type, recommended action, and auto-remediation suggestion

Jurisdictions screened: **ZA_JSE**, **US_SEC**, **UK_FCA**, **EU_ESMA**, **global**

Risk types detected: material_non_public, forward_looking, insider_trading, selective_disclosure, market_manipulation

### `generateAutoDraft(questionText, eventName, clientName, category)`

Generates a compliance-safe draft response:

- Under 200 words
- Avoids forward-looking statements
- Includes appropriate disclaimers
- Flags if management/legal input needed
- Returns `answerText` + `reasoning`

## 6. Operator Console UI

**File**: `client/src/components/LiveQaDashboard.tsx` (578 lines)
**Tab**: "Live Q&A" in Shadow Mode (8th tab)

### Top Bar
- Event title + session code
- Live question count
- **Share Link** button (copies `/qa/{code}` to clipboard)
- **Pause Q&A** / **Resume Q&A** toggle
- **End Q&A** button
- **Show/Hide Insights** toggle

### Main Panel — Live Question Queue
Each question renders as a `QuestionCard` with:

- **Triage Score Badge**: HIGH (red) / MED (amber) / LOW (green) with numeric priority
- **Status Badge**: Pending / Triaged / Approved / Answered / Rejected / Flagged
- **Category Badge**: financial / operational / esg / governance / strategy / general
- **Compliance Risk Indicator**: GREEN (clear) / AMBER (medium) / RED (high risk)
- **Submitter Info**: Name, company (hidden if anonymous)
- **AI Triage Reason**: Italic explanation from the triage engine
- **Upvote Count** and **Unresolved Compliance Flags** count

**One-Click Actions** (per question):

| Button | Action | Effect |
|--------|--------|--------|
| **Approve** | Sets status to `approved` | Question visible to attendees, ready for speaker |
| **Route to Bot** | Generates AI draft + sets approved | Opens answer panel with pre-filled AI response |
| **Legal Review** | Sets status to `flagged` | Adds operator note "Escalated for Legal Review" |
| **Reject** | Sets status to `rejected` | Removes from attendee view |
| **Generate AI Draft** | Calls `generateDraft` | Opens answer panel with AI-generated response |
| **Respond** | Expands answer panel | Operator types/edits response, submits |

### Right Sidebar — Predictive Insights

| Panel | Content |
|-------|---------|
| **Session Analytics** | Total questions, approved count, flagged count, average triage score |
| **Live Sentiment** | Real-time compliance risk level (Positive/Neutral/Cautious) with risk bar |
| **Materiality Heatmap** | Category distribution with proportional bars |
| **Anticipated Questions** | Top 5 predicted analyst questions with match detection (crossed out when asked) |

### Status Filter Bar
Filterable tabs: All / Pending / Triaged / Approved / Answered / Rejected / Flagged — each with count

### Launch Screen (no active session)
- "Launch Live Q&A Session" button
- Previous sessions list with status badges

## 7. Attendee Q&A Interface

**File**: `client/src/pages/AttendeeQA.tsx` (224 lines)
**Route**: `/qa/:accessCode` — **public, no login required, mobile-friendly**

### Features
- Live event header with session status indicator (pulsing green = live)
- Question submission form (textarea + optional name/company/email + anonymous toggle)
- Character limit: 5–2000 characters
- Real-time question list (5s polling) showing approved/triaged/answered questions
- Upvote button per question (local dedup via votedIds state)
- Category badges and "Answered" status
- Paused/Closed session messaging
- Footer: "Powered by CuraLive"

### States
- **Loading**: Spinner while fetching session
- **Invalid Code**: "Invalid Q&A Link" message
- **Session Not Found**: Error state
- **Active Session**: Full submission form + question list
- **Paused Session**: Info banner, no submission form
- **Closed Session**: End message, questions still visible

## 9. Security Model

| Layer | Implementation |
|-------|----------------|
| Operator auth | All operator endpoints use `operatorProcedure` (requires `operator` or `admin` role) |
| Public endpoints | `submitQuestion`, `listQuestionsPublic`, `getSessionByCode`, `upvoteQuestion` only |
| Upvote abuse prevention | 10-second cooldown per fingerprint, auto-purge of stale entries |
| Insert ID safety | Uses MySQL `insertId` from INSERT result (no race condition) |
| SQL injection | All dynamic queries use parameterized `conn.execute(query, params)` |
| Data exposure | Public list only shows triaged/approved/answered questions (hides pending/rejected/flagged) |
| Anonymous submissions | Name and company stripped at insert time when `isAnonymous = true` |

## 10. Connection Flow (15-second start)

1. Operator logs into `company.curalive.app`
2. Goes to **Shadow Mode** tab
3. Clicks **Live Q&A** tab
4. Clicks **Launch Live Q&A Session** (auto-links to active Shadow Mode session if one exists)
5. System generates 8-char session code and creates Q&A session
6. Operator clicks **Share Link** — copies `company.curalive.app/qa/KV3N8P2Q` to clipboard
7. Sends link to attendees via chat/email/meeting chat
8. Questions flow in with AI triage in real-time

No extra logins. No separate apps. No platform dependencies.
